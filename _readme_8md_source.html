1 README
2 ========================================
3 A C++ component-based game engine, built using [RIKU]. This engine is currently
4 intended for small-scale game prototyping, but will eventually be extended to a
5 more general-purpose game engine and editor. Also included is the tiny_core
6 library, which uses [ttvfs] and [jsoncpp] to provide default filesystem and
7 serializer implementations.
8 
9 **Features**
10  - Introspective
11  - Data driven
12  - Component based
13  - Highly extensible

26 ### Build with CMake
27 
28 ```bash
29 cd /path/to/build/folder
30 cmake /path/to/tinyeng
31 cmake /path/to/tinyeng --build
32 ```
33 
34 As part of the build process, CMake will download additional libraries. At a
35 minimum [RIKU] will be downloaded adjacent to the `tinyeng` source folder. If
36 the `TINY_CORE` CMake flag is enabled (which it is by default), [ttvfs] will be
37 also be downloaded to `ext/ttvfs`, next to both `RIKU` and `tinyeng`.
38 
39 
40 ### Add to an existing CMake project
41 
42 ```cmake
43 add_subdirectory("/path/to/tinyeng" "${CMAKE_CURRENT_BINARY_DIR}/tiny_build")
44 
45 target_link_libraries(your_library tiny)
46 ```
47 
48 If you want to use the tiny core library, link `tiny_core` instead of `tiny` with
49 `target_link_libraries()`. That will automatically link `tiny` as well.
50 
51 
52 ### Link statically
53 
54 Build and add tinyeng.lib from the `build` folder to your project, and add
55 `tinyeng/include` to the list of include directories.
